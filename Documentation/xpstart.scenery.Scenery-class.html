dataFile

The file were the data of the scenerie is cached That parameter is overwritten In one file there can be stored more data. The logic for storing is classname:title:dataname:data

sceneryTxt

The scenery.txt is a place where information about layergroup can be defined. This gives the developer the possibility to define when his scenery should be loaded by the system.

icaoCodes

Icao codes of the scenery if there is an apt.dat One scenery can have many icao codes. The parameter is type list
